# Vision（愿景与边界）

## 使命（1 句话）

把科研中的“可机械化部分”变成可执行流水线：从文献 → 推导/代码 → 数值/复现 → 审稿/修改 → 写作/发布，每一步都有证据、日志与可复现产物。

## 为什么要做（痛点）

- 科研耗时往往不在“想法”，而在重复劳动：查文献、抄公式、写脚本、调数值稳定性、整理图表、补引用、按审稿意见反复修改。
- LLM 能“写东西”，但科研要“能跑、能证、能复核”：需要工具调用、产物契约、审计与门禁，而不仅是对话。

## 北极星能力（North-star capabilities）

按“可落地、可验收”优先级排序：

1) **文献与证据入口**：给 INSPIRE recid / arXiv / DOI → 自动抓源文件与元数据 → 生成可追溯阅读笔记（带RefKey、链接、要点、可疑点）。
2) **复现主结果（reproduction-first）**：从论文中锁定 1–3 个“主结果”（图/表/数值）→ 自动生成复现计划 → 执行计算 → 对比误差与差异来源 → 输出复现报告与 artifacts。
3) **自动审稿与修改闭环**：对 LaTeX 工程生成结构化审稿意见（major/minor）→ 生成 revision plan → 自动改稿（带 diff、证据门禁、可编译）→ 再审直到收敛或触发 kill criteria。
4) **可审计的自动推导/自动代码**：把“可重复验证的推导步骤”写入 notebook（不跳步）并与代码符号/产物绑定；能自动做极限/维度/规范不变性/交叉检查。
5) **多用户可用**：把“研究习惯”产品化成入口（CLI/Web），对外提供稳定的权限模型、日志、版本锁定与可复现环境。

## 非目标（Anti-goals）

为了避免过度承诺，明确不做/暂不做：

- 不承诺“无需人类监督自动做出新物理发现”。短期目标是**加速与降错**，不是替代判断。
- 不把“幻觉式总结”当成果；任何关键结论必须指向可复现证据（artifacts 或推导/检查）。
- 不在未知/不受控环境里执行不可信代码（默认只跑本地/受控工具链；对网络/安装/执行有门禁）。

## 设计原则（Design principles）

1) **Evidence-first**：关键结论必须可追溯到 artifacts 或可复核推导。
2) **Reproducibility by default**：每次运行必须落盘：manifest/summary/analysis + 环境快照 + 日志。
3) **多代理 = 分工 + 约束 + 复核**：Planner/Runner/Writer/Reviewer 等角色隔离上下文与工具权限。
4) **渐进式自动化**：先把“高ROI、低风险”的环节自动化（抓取/整理/门禁/复现），再扩展到推导与创新辅助。
5) **模型可替换**：Codex/Claude/Gemini 只是“大脑”，系统价值在于：工具、契约、评测、复核与产物。

## 成功指标（可量化）

- 同一类论文复现任务：人类干预次数、总耗时、复现通过率、差异定位时间。
- 写作与修改：一次 revision 周期的平均回合数、LaTeX 可编译率、引用完整率、证据门禁通过率。
- 可靠性：回归评测（eval suite）在升级后不退化。

## 长期愿景：多 Agent 自主研究社区 (Agent-arXiv)

> 本节描述的是**远期目标** (Phase 5+)，当前阶段的所有架构决策应兼容此方向，但不以此为近期交付物。

### 愿景描述

在"人类使用的研究加速工具"之上，建立一个**多 Agent 自主研究的实验社区**：

1. **文献池启动**: 以 hep-th arXiv 库 (或其子集) 作为初始知识库，导入大量文献
2. **Agent 自主选题**: 多个 Agent 基于自身能力和文献池中的研究空白，自由挑选课题
3. **独立并行研究**: 每个 Agent (或 Agent 团队) 独立进行课题研究——检索、推导、计算、验证、撰写
4. **结果发布与引用**: 研究完成后，Agent 将结果发布到内部"Agent-arXiv"，其他 Agent 可引用
5. **知识积累迭代**: 已发布的研究成果成为后续研究的基础，形成迭代的知识增长
6. **进化观测**: 观察 Agent 社区从初始文献出发，经过 N 轮自主研究后，知识前沿的推进程度

### 核心目的

这不是"替代人类科学家"，而是一个**受控实验**：

- 测试当前 AI Agent 在理论物理科研中的**自主能力边界**
- 发现 Agent 科研的系统性弱点 (如创新性不足、循环论证、计算可靠性)
- 为 Agent-human 协作研究提供数据基础
- 生成真实的负面/正面案例库，反馈到 idea-core 的进化策略

### 设计约束

1. **Evidence-first 不可妥协**: Agent 发布的"论文"必须通过与人类论文相同的证据门禁
2. **科学诚信优先**: EVO-06 诚信框架在 Agent-arXiv 中为 fail-closed (无诚信报告不可发布)
3. **人类可审计**: 所有 Agent 研究过程有完整 trace，人类可随时审查任何"论文"的完整推导链
4. **资源有限性**: Agent 社区在固定 token/compute 预算下运行，强制资源效率竞争
5. **可关停**: 任何时候人类可冻结整个社区，审查所有已发布结果

### 成功指标

- **知识覆盖**: 从 N 篇种子论文出发，Agent 社区能将多少 % 的核心结果独立复现
- **原创贡献**: Agent 产出中有多少条"真正新的 idea"通过人类专家评审
- **引用网络健康度**: 内部引用网络的连通性、深度、循环引用率
- **失败学习率**: 重复相同失败的频率是否随迭代下降 (EVO-09 + EVO-10 的验收指标)
- **计算效率**: 每单位 token/compute 预算产出的已验证新知识量
