## 3. Prior blocker closure from v1 review

- B1 (operator selection entry point): `search.step` now selects operators only via `_choose_search_operator`; `_next_search_operator` is the round-robin helper used from that entry point only.
- B2 (SearchOperator protocol uncertainty): `SearchOperator` is a `Protocol` and is now runtime-checkable; added a unit test to prove the M3.2 operators conform.
- B3 (seed_count validation ordering): `initial_island_count > seed_count` now fails before campaign dict construction.
- B4 (_refresh_island_population_sizes missing): helper already exists in this file; added an inline comment at the call site for clarity.

## 4. Acceptance mapping

1. Operators: >=3 families in HEP DomainPack
   - Added `hep.operators.v1` DomainPack with 3 deterministic operators:
     - family=`AnomalyAbduction` (operator_id=`hep.anomaly_abduction.v1`)
     - family=`SymmetryOperator` (operator_id=`hep.symmetry_operator.v1`)
     - family=`LimitExplorer` (operator_id=`hep.limit_explorer.v1`)
2. Different islands produce different node “styles”
   - `operator_selection_policy="island_index_v1"` maps each `island-N` deterministically to operator `N % 3`.
3. Repopulation does not collapse diversity
   - Multi-island state machine triggers `STAGNANT -> REPOPULATED`; selection remains island-indexed so families stay distinct across repopulation ticks.
4. Multi-island campaign seeding for operator execution
   - Added charter extension `initial_island_count` (or `island_count`) to initialize islands in `campaign.init`.
   - Seeds are distributed round-robin across islands; fail-fast if `initial_island_count > seed_count`.
