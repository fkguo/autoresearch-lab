1. WorkOrder artifact + ledger event
   - Added `WorkOrder` typed payload and `record_work_order()`.
   - Persists JSON artifact under `artifacts/work_orders/<work_id>.json`.
   - Appends ledger event `work_order.created` with `work_id`, `idempotency_key`, `artifact_ref`, `artifact_hash`.

2. WorkResult artifact + ledger event
   - Added `WorkResult` typed payload with status guard (`ok|failed|canceled|budget_exhausted|permission_denied`).
   - Persists JSON artifact under `artifacts/work_results/<work_id>.json`.
   - Appends ledger event `work_result.recorded` with `work_id`, `status`, artifact provenance fields.

3. TeamPlan artifact + ledger event
   - Added `TeamPlan` typed payload with coordination-policy guard (`parallel|sequential|stage_gated`).
   - Persists JSON artifact under `artifacts/team_plans/<team_id>.json`.
   - Appends ledger event `team_plan.registered` with `team_id`, `coordination_policy`, artifact provenance fields.

4. Ledger append-only & auditability
   - Added `append_ledger_event()` and `read_ledger_events()`.
   - All three write paths emit timestamped, UUID-backed events into `ledger/events.jsonl`.
   - Artifact hash uses canonical `sha256` payload hash (`payload_hash`).
