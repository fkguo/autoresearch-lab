## 3. Acceptance mapping (M4.2)

1. WorkOrder -> session + role execution message
   - Added `OpenCodeRuntimeAdapter.execute_work_order()`.
   - Uses `create_session(work_id, input_artifacts, tool_policy)` and `post_message(session_id, role_message)`.
   - Writes ledger event `runtime.session_started` and persists `WorkResult` artifact.
   - Adds failure-path ledger event `runtime.session_failed` when runtime client calls fail.

2. Permission handshake -> gate event + runtime response
   - Added `handle_permission_request()`.
   - Captures runtime permission request into ledger (`runtime.permission_requested`).
   - Applies gate decision + policy guard and sends response through `post_permission(...)`.
   - Writes resolution event `runtime.permission_resolved`.
   - Adds failure-path ledger event `runtime.permission_failed` when permission callback fails.

3. SSE observability -> ledger entries with artifact hash
   - Added `ingest_sse_events()` consuming `stream_global_events(session_id=...)`.
   - Persists `permission_request` and `execution_summary` events into ledger.
   - Emits deterministic `artifact_hash` (`payload_hash`, RFC8785 canonical SHA-256) for observed runtime payloads.
   - Adds replay safety via persisted ledger dedup key check (`has_ledger_event(session_id,event_key)`), so replayed SSE events are skipped across restarts.
   - Adds stream guards `max_events` + `max_wall_s` with guard event `runtime.sse_limit_reached`.
   - Uses producer thread + queue timeout so wall-clock guard remains effective even on blocking streams.
   - Adds fixture-based contract test (`opencode_sse_session_v1.jsonl`) to pin expected SSE event shapes.

4. Tool policy: default deny + high-risk gated + fs whitelist
   - Added `ToolPolicyEnforcer` with `is_tool_allowed()`.
   - High-risk tools (`shell/fs/mcp`) require gate approval.
   - Default deny if tool not in allow-list.
   - `fs` additionally requires canonicalized requested path (`resolve(strict=False)`) under configured `write_roots` whitelist, fail-closed on invalid paths.
