## What is new in W6-30 (vs W6-29)?

W6-29 tightened the $Q^2\\in[0,2]\,\\mathrm{GeV}^2$ band by adding a high-$Q^2$ **value-band UV anchor** on $A^\\pi(-Q^2)$ at $Q^2=10\\,\\mathrm{GeV}^2$, using LO pQCD (arXiv:2412.00848 Eq. `Qem`) and an explicit but ad hoc **$\\pm 50\\%$** tolerance proxy.

W6-30 upgrades that “$\\pm 50\\%$ knob” into an **auditable uncertainty budget artifact** that can be **gate-enforced** and **config-bound**:

- New schema: `idea-runs/schemas/uv_anchor_budget_v1.schema.json`
- New builder: `idea-runs/scripts/build_uv_anchor_budget_v1.py`
- New gate enforcement (in `idea-runs/scripts/validate_project_artifacts.py`):
  - If any compute config uses `constraints.low_energy_value_bands`, project must contain `artifacts/assumptions/uv_anchor_budget_v1.json` (schema-validated).
  - If any compute config sets `constraints.uv_anchor_budget_binding=true`, then:
    - budget must be `budget_mode=derived`;
    - for each anchor, `absolute_tolerance == \\sum_i (budget_components[i].abs_contribution)`;
    - and the compute config’s `(Q2_mpi2, A_target, absolute_tolerance)` must match the budget **exactly** (no silent drift / no “pretend physics”).

Goal: keep the physics input explicit (still a proxy, but now budgeted), while preventing silent hand-tuning of UV tolerances.

Scope remains pion-only; no coupled-channel.
