W6-08 erratum (2026-02-17): v19 Q* Clarabel vs ECOS are NOT identical at solver-input level

Context:
- In W6-08 Round-001 packet + summary-v1, we described the v19 Clarabel and v19 ECOS single-point Q* runs as “identical conic program; only solver differs”.
- That wording is too strong for JuMP/MathOptInterface workflows: MOI applies solver-dependent bridges, so the *inner* conic form can differ even when the JuMP-level constraints and config JSON are the same.

Evidence (v19):
- Inner model stats differ (notably Nonnegatives representation):
  - docs/reviews/bundles/2026-02-16-w6-08-v19-inner-model-stats-v1.txt
- Fingerprint differs across solvers (SHA256 mismatch), despite the same JuMP-level config:
  - docs/reviews/bundles/2026-02-16-w6-08-v19-inner-fingerprint-v1.txt

Implication:
- The v19 Clarabel/ECOS objective spread at Q* cannot be interpreted as “same conic program solved differently”.
- We therefore treat the ECOS run as a *robustness probe* (same physics inputs + same high-level discretization intent), not as a strict identical-program cross-solver certificate.
