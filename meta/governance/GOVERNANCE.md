# Governance

## Ownership

- Org owner: fkguo
- Current mode: single-owner private operation

## Decision Rules

- Compatibility changes require updates to `ecosystem-manifest.json`.
- Breaking changes must increment version and state migration notes.

## Release Rules

- `dev`: internal experiments only
- `beta`: integration-ready, evolving
- `stable`: production-facing and backward-compatible

## Deferred Policies

The following are intentionally deferred until team expansion:
- Team-based permissions
- Mandatory protected branches
- CODEOWNERS-based approval routing
