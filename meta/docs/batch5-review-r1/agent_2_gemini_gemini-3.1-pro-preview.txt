### NEW-02: Approval Packet Trio

#### [BLOCKING] Missing Directory Creation in `write_trio`
In `packages/hep-autoresearch/src/hep_autoresearch/toolkit/approval_packet.py`, the `write_trio` function attempts to write files to a path (typically inside an `approvals/` subdirectory). If the subdirectory does not exist, `open(path, "w")` will raise a `FileNotFoundError`.
*   **Fix**: Add `os.makedirs(os.path.dirname(base_path), exist_ok=True)` before writing.

#### [NON-BLOCKING] Schema Strictness
The `meta/schemas/approval_packet_v1.schema.json` is well-structured, but it lacks `"additionalProperties": false` at the top level. To ensure the SSOT (Single Source of Truth) remains clean and codegen doesn't produce unexpected types in TypeScript, it is recommended to restrict additional properties.

#### [NON-BLOCKING] `render_short` Truncation Logic
The implementation uses `lines[:57]` to enforce the 60-line limit. While functional, if the `gate_resolution_trace` itself contains very long lines that wrap in a standard terminal, the "visual" line count will exceed 60. This is acceptable for now given the "zero external dependency" constraint, but consider a more robust wrapping check in the future.

### NEW-03: Approvals CLI Show Command

#### [NON-BLOCKING] UX: Missing Run-ID Feedback
In `cmd_approvals_show`, if a user provides a `--run-id` that exists but has no approval packets, the CLI returns 0 with an info message. While this matches the requirement, it is helpful to distinguish between "Run not found" and "Run found, but no approvals recorded." Currently, it treats both similarly.

### NEW-04: Self-Contained Human Report

#### [BLOCKING] Security/Integrity: Binary File Reading for SHA256
In `packages/hep-autoresearch/src/hep_autoresearch/toolkit/report_renderer.py`, the `collect_run_result` function computes SHA256 hashes for artifacts.
*   **Issue**: If files are opened in text mode (`"r"`), the hash will vary based on the system's default encoding and newline handling (CRLF vs LF), breaking the audit trail.
*   **Fix**: Ensure all files used for hashing are opened in binary mode: `with open(path, "rb") as f:`.

#### [NON-BLOCKING] LaTeX Escaping Completeness
The `_tex_escape` function handles standard characters (`&`, `%`, `$`, etc.). However, it should also handle backslashes (`\`) by converting them to `\textbackslash{}` and tildes (`~`) to `\textasciitilde{}`. Without this, a skill output containing a file path like `C:\Users\~` will break the LaTeX compilation.
